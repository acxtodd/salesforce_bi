1
Salesforce CDC Platform Event Published
A user creates a new Property record. Salesforce automatically publishes an ascendix__Property__ChangeEvent on the CDC channel with changeType CREATE, the new record ID, and a commit timestamp.
{ "ChangeEventHeader": { "entityName": "ascendix__Property__ChangeEvent", "changeType": "CREATE", "recordIds": ["a01xx00000ABC123"], "commitTimestamp": 1710720000000 }, "Name": "One Arts Plaza", "ascendix__City__c": "Dallas" }
Trigger: automatic on DML
2
AppFlow CDC Stream → S3
AWS AppFlow polls the CDC channel using a JWT Bearer connector profile (credentials in Secrets Manager). On detection, it writes the raw JSON payload to S3.
s3://salesforce-ai-search-cdc-{acct}-{region}/ cdc/ascendix__Property__c/ 2026/03/18/1710720060000.json
Latency: 1-5 min polling Auth: JWT Bearer lib/ingestion-stack.ts:105-225
3
S3 Object Created Notification
The CDC bucket has EventBridge notifications enabled. When AppFlow writes the JSON file, S3 emits an Object Created event to EventBridge.
Latency: <1s enableEventBridgeNotification()
4
EventBridge Rule Match → Lambda Target
Rule salesforce-ai-search-cdc-trigger matches any S3 key prefixed cdc/. It transforms the event into a flat payload and invokes the CDC Sync Lambda.
// Input transform { "bucket": "salesforce-ai-search-cdc-...", "key": "cdc/ascendix__Property__c/2026/...", "eventTime": "2026-03-18T...", "eventSource": "cdc" }
Latency: <1s lib/ingestion-stack.ts:965-1014
5
Lambda Parse CDC Payload
The Lambda reads the S3 object, parses ChangeEventHeader, and maps the entity through CDC_ENTITY_MAP: ascendix__Property__ChangeEvent → ascendix__Property__c. Extracts changeType and recordIds.
CDC_ENTITY_MAP = { "ascendix__Property__ChangeEvent": "ascendix__Property__c", "ascendix__Property__c": "ascendix__Property__c", ... } # Result: CDCEvent(object_name, change_type, record_ids, commit_ts)
lambda/cdc_sync/index.py:57-91
6
Lambda + Salesforce API SOQL Enrichment Query
CDC payloads only carry changed fields. The Lambda queries Salesforce for the complete record using a SOQL built from denorm_config.yaml, including parent lookups for OwnerLandlord, Market, and SubMarket.
SELECT Id, LastModifiedDate, Name, ascendix__City__c, ascendix__State__c, ascendix__PropertyClass__c, ascendix__Description__c, ascendix__TotalBuildingArea__c, ascendix__YearBuilt__c, OwnerLandlord.Name, Market.Name, SubMarket.Name FROM ascendix__Property__c WHERE Id = 'a01xx00000ABC123' LIMIT 1
Latency: 1-5s REST API v59.0 lib/denormalize.py:85-115
7
Denormalize Flatten & Build Searchable Text
flatten() unpacks parent relationships into flat keys. build_text() constructs a pipe-delimited string from all embed fields — this becomes the BM25-indexed text and the embedding input.
// flatten() output { "Name": "One Arts Plaza", "ascendix__City__c": "Dallas", "ascendix__State__c": "TX", "ascendix__OwnerLandlord__c": {"Name": "Ascendix LLC"}, "ascendix__Market__c": {"Name": "Uptown"} } // build_text() output "Property: Name: One Arts Plaza | City: Dallas | State: TX | PropertyClass: A | OwnerLandlord Name: Ascendix LLC | Market Name: Uptown | SubMarket Name: Arts District"
Latency: <100ms lib/denormalize.py:123-192
8
Bedrock Generate Vector Embedding
The full text string is sent to Amazon Titan Embed Text v2 via Bedrock, returning a 1024-dimension normalized vector for semantic search.
bedrock.invoke_model( modelId="amazon.titan-embed-text-v2:0", body={ "inputText": "Property: Name: One Arts Plaza | ...", "dimensions": 1024, "normalize": True } ) # → [0.0234, -0.0891, 0.0412, ... ] (1024 floats)
Latency: 1-2s 1024 dimensions Cosine distance lib/denormalize.py:293-307
9
Turbopuffer Upsert Document
The final document is upserted into Turbopuffer namespace org_{salesforce_org_id}. The text field is indexed with word_v3 tokenizer for BM25. The vector is indexed for ANN. All metadata stored as filterable attributes.
{ "id": "a01xx00000ABC123", "vector": [0.0234, -0.0891, ...], // 1024-dim "text": "Property: Name: One Arts...", // BM25 indexed "object_type": "property", "name": "One Arts Plaza", "city": "Dallas", "state": "TX", "propertyclass": "A", "totalbuildingarea": 250000, "yearbuilt": 2007, "ownerlandlord_name": "Ascendix LLC", "market_name": "Uptown" }
Latency: <1s Schema: word_v3 tokenizer lib/turbopuffer_backend.py:313-329
✓ Record is now searchable
10
Search Hybrid Query (BM25 + ANN + RRF)
At query time, the search backend runs BM25 and vector ANN in parallel via multi_query, then fuses results client-side using Reciprocal Rank Fusion (k=60). Attribute filters narrow the scope.
// multi_query: two parallel searches Query 1 (BM25): rank_by=("text", "BM25", "arts plaza dallas") Query 2 (ANN): rank_by=("vector", "ANN", embed("arts plaza dallas")) // RRF fusion score[doc] = Σ 1/(60 + rank_bm25) + 1/(60 + rank_ann) // Result [{id: "a01xx00000ABC123", name: "One Arts Plaza", dist: 0.92, ...}]
3 modes: ANN, BM25, Hybrid Filterable attributes lib/turbopuffer_backend.py:163-211
!
Error Path DLQ + CloudWatch Metrics
If any step (5-9) fails, the event is sent to salesforce-ai-search-ingestion-dlq (14-day retention) with the error and raw payload. On success, CDCSyncLag is emitted to CloudWatch per SObject.
// DLQ message on failure { "object_name": "ascendix__Property__c", "change_type": "CREATE", "record_ids": ["a01xx00000ABC123"], "error": "SalesforceQueryError: INVALID_SESSION_ID", "raw": { ... } } // CloudWatch metric on success Namespace: SalesforceAISearch/CDCSync Metric: CDCSyncLag = 4200 ms Dimension: SObject = ascendix__Property__c
DLQ retention: 14 days lambda/cdc_sync/index.py:315-378
