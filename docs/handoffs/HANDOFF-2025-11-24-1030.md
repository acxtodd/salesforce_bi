## Changes Implemented

### 1. Architectural Shift: Docker + Web Adapter
- **Moved from**: Zip-based Python Lambda with native handler.
- **Moved to**: Docker-based Lambda using `public.ecr.aws/docker/library/python:3.11-slim`.
- **Middleware**: Added `aws-lambda-adapter` (v0.8.4) to proxy API requests to a local web server.
- **Framework**: Implemented `lambda/answer/main.py` using **FastAPI** to handle streaming responses via `StreamingResponse`.

### 2. Infrastructure Updates (CDK)
- **ApiStack**: Updated `AnswerLambda` to use `lambda.DockerImageFunction` with `Architecture.ARM_64`.
- **MonitoringStack**: Temporarily detached and then re-attached to resolve CloudFormation circular dependency during the migration.
- **Function URL**: Enabled Function URL with `INVOKE_MODE=RESPONSE_STREAM` for the Answer Lambda.

### 3. Code Fixes
- **Dependencies**: Fixed `ImportError` by building dependencies inside Docker (Linux environment) instead of locally on Mac (Darwin).
- **Entrypoint**: Switched to `python:3.11-slim` base image to avoid conflicts with AWS Lambda's default `bootstrap` entrypoint.
