# Archive Guide

Files under `docs/archive/` are historical unless a task explicitly says
otherwise.

Use archive docs for:

- migration archaeology
- postmortems
- legacy AWS decommission work
- comparing the old graph/Bedrock stack against the current connector

Do not use archive docs as the source of truth for:

- current architecture
- current object scope
- current sync ownership
- current operator procedures
- current acceptance status

For current-state guidance, start with:

1. `README.md`
2. `docs/architecture/object_scope_and_sync.md`
3. `TASK_TRACKING.md`
4. `CLAUDE.md`
